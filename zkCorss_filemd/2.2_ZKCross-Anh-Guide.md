# ZKCross v2 — Hướng dẫn chạy đầy đủ theo ZKCross-Anh

Hướng dẫn này tổng hợp và chi tiết hóa 4 kịch bản chính trong `ZKCross-Anh.pdf` dựa trên các script thực tế hiện có trong `scripts/`.

## Mục tiêu

Hoàn thành đầy đủ 4 kịch bản được yêu cầu:

1. Scenario 1: Network latency (VM)
2. Scenario 2: Sepolia deployment (local)
3. Scenario 3: Groth16 RAM benchmark (local)
4. Scenario 4: MF-PoP reputation simulation (local)

## Tổng quan các script chính

| Kịch bản | Script | Chạy ở đâu |
| --- | --- | --- |
| Scenario 1 | `scripts/network_latency_experiment.sh` | VM
| Scenario 2 | `scripts/deploy_sepolia.cjs` | Local
| Scenario 3 | `scripts/groth16_ram_benchmark.cjs` | Local
| Scenario 4 | `scripts/mfpop_simulation.py` | Local

## Các script hỗ trợ bổ sung có thể dùng tiếp

| Script | Mục đích |
| --- | --- |
| `scripts/deploy_to_10vm.sh` | Deploy toàn bộ dự án lên 10 VM từ local
| `scripts/run_all_vms_experiments.sh` | Chạy experiments trên tất cả 10 VM và thu kết quả
| `scripts/run_azure_experiments.sh` | Chạy TN2/TN3/TN4 trên một VM
| `scripts/run_all_experiments.sh` | Chạy local + VM experiment tổng hợp

Nếu bạn đã chạy `deploy_to_10vm.sh` và `run_all_vms_experiments.sh`, thì bạn đã sử dụng được phần lớn các bước VM cần thiết. Hướng dẫn dưới đây sẽ kết hợp các bước đó và bổ sung phần local còn thiếu để đảm bảo hoàn chỉnh.

## 1. Chuẩn bị môi trường

### 1.1. Trên máy local

```bash
cd /home/mihwuan/Project/zkCross
npm install
python3 -m pip install matplotlib numpy --break-system-packages
```

### 1.2. Trên mỗi VM

Cần có:
- Docker
- Docker Compose
- `tc` / `iproute2`
- SSH với tài khoản `azureuser`

Nếu bạn chạy `scripts/deploy_to_10vm.sh`, script này đã đảm bảo:
- sync dự án lên 10 VM
- cài `node`, `npm`, gỡ nvm và cài Node 22
- chạy `npm install` trên mỗi VM
- start Docker compose trên mỗi VM
- cấu hình mạng nội bộ cho 10 VM

Vậy nếu `deploy_to_10vm.sh` chạy thành công, phần chuẩn bị VM cơ bản đã được đáp ứng.

## 2. Chuẩn bị 10 VM bằng `deploy_to_10vm.sh`

### 2.1. Chạy script deploy

```bash
cd /home/mihwuan/Project/zkCross
bash scripts/deploy_to_10vm.sh
```

### 2.2. Script này thực hiện

- sync project lên các VM
- chạy `azure_vm_setup.sh` trên mỗi VM
- cài Node.js 22.x và `npm install`
- copy `docker-compose-10vm.yml` và `.env.vmN`
- khởi chạy Docker container trên mỗi VM
- compile contracts trên mỗi VM
- thiết lập mạng nội bộ cho 10 VM

### 2.3. Khi thành công

Bạn sẽ có sẵn:
- 10 VM đã sync project
- 10 VM chạy Docker với 10 chains mỗi VM
- 10 VM đã compile contract
- hệ thống mạng giữa các VM được cấu hình

Lưu ý: script không tự động chạy experiment Scenario 1, chỉ chuẩn bị VM và Docker.

## 3. Scenario 1: Network latency (VM)

### 3.1. Điều kiện cần

- VM đã có Docker và 20 container
- `tc` có thể sử dụng trong các container
- `docker compose up -d` đã chạy thành công trên mỗi VM

### 3.2. Chạy experiment

Trên mỗi VM:

```bash
cd ~/zkCross
sudo bash scripts/network_latency_experiment.sh
```

### 3.3. Output cần có

- `results/network_latency/baseline.json`
- `results/network_latency/latency_50ms.json`
- `results/network_latency/latency_150ms.json`
- `results/network_latency/latency_300ms.json`

### 3.4. Ghi chú

Nếu hệ thống khối không chạy đủ container, `network_latency_experiment.sh` sẽ không đo chính xác.

## 7. Sử dụng `run_all_vms_experiments.sh`

Nếu bạn đã chuẩn bị VM từ `deploy_to_10vm.sh`, thì `run_all_vms_experiments.sh` sẽ giúp chạy experiment trên tất cả VM và thu dữ liệu về local.

### 7.1. Chạy script

```bash
cd /home/mihwuan/Project/zkCross
bash scripts/run_all_vms_experiments.sh
```

### 7.2. Script này thực hiện

- kiểm tra SSH tới 10 VM
- kiểm tra Docker status
- deploy contract nếu chưa có `deployment_v2.json`
- chạy `bash scripts/run_azure_experiments.sh` trên từng VM
- copy kết quả VM về `results/all_vms/vmN`
- tạo `results/all_vms/summary.json`

### 7.3. Nhược điểm

- `run_all_vms_experiments.sh` không chạy Scenario 1 (network latency)
- `run_all_vms_experiments.sh` không chạy local Scenario 2/3/4

Vậy, nếu bạn cần hoàn chỉnh 4 kịch bản, phải kết hợp thêm:
- `scripts/network_latency_experiment.sh` cho Scenario 1
- `scripts/deploy_sepolia.cjs` cho Scenario 2
- `scripts/groth16_ram_benchmark.cjs` cho Scenario 3
- `scripts/mfpop_simulation.py` cho Scenario 4

## 12. Xử lý khi có vấn đề

- Thiếu `node`: cài Node.js
- Thiếu `python3`: cài Python 3
- Thiếu gói Node: `npm install`
- Thiếu gói Python: `python3 -m pip install matplotlib numpy`
- Nếu `deployment_v2.json` không tồn tại: chạy `VM_ID=N node scripts/deploy_contracts_v2.cjs`
- Nếu `network_latency_experiment.sh` không chạy: kiểm tra Docker, container và `tc`
