## Kịch bản 4: MF-PoP Reputation Simulation

### Trạng thái: ✅ HOÀN THÀNH

### Mục đích
Mô phỏng hệ thống danh tiếng (MF-PoP) dưới điều kiện Byzantine, xác nhận:
1. Oscillating attack không thành công
2. Byzantine committers bị cách ly trong ~48 rounds
3. Honest committers vẫn được reward

### Kết quả có được
```
/home/mihwuan/Project/zkCross/results/
├── mfpop_simulation_data.json       # Raw data từ simulation
├── mfpop_reputation_recovery.png    # Hình: Reputation convergence
├── mfpop_stake_slashing.png         # Hình: Stake slashing
└── (old) mfpop_analysis.py          # Analysis script

Key metrics:
- Attacker reputation @ R200: 0.01 ✅ (isolated)
- Honest reputation @ R200: 0.9999 ✅ (maintained)
- Isolation round: 48.0 ± 0.0 ✅ (matches Theorem 1)
- Total slashed: 96.9% ✅ (matches Table VI)
- Accuracy recovery: 70% → 100% @ R46
```

### Phương pháp triển khai
- **Language:** Python 3
- **Simulation:** 200 rounds, 11 committers (10 honest, 1 Byzantine)
- **Attack pattern:** 5 correct proofs + 1 incorrect (oscillating)
- **Parameters:** 
  - Beta (decay): 0.08
  - Slash multiplier: 0.5
  - Trust Jail: R < 1.5 × R_min

### Kết quả chi tiết
```
Stats từ 10 seeds (runs):
- Mean isolation round: 48.0 ± 0.0
- Attacker final R: 0.01 (100% consistency)
- Honest final R: 0.9999 (>99.99%)
- Accuracy final: 100% (after isolation)
- Stake slashed: 96.9% average

Comparison (Old vs New MF-PoP):
                     | Old (γ=0.7, β=0.3) | New (γ=0.7, β=0.08 + slash)
Attacker R @ R200    | 0.84 (NOT isolated) | 0.01 (isolated) ✅
Stake remaining      | ~100% (problem!)   | 3.1% (slashed 96.9%) ✅
Weight separation    | 1.4× (weak)        | 7236× (strong) ✅
```
